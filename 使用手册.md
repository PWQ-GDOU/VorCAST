## 2. 数据准备

### 2.1 需要的数据集

| 数据集 | 格式 | 说明 | 时空分辨率 |

| **HRRR 分析场**（可选） | `.nc` (NetCDF) | 高分辨率快速刷新模式输出 | 1 小时，3 km 水平，36 层气压 |

### 2.2 目录结构要求

```
数据根目录/
├── nexrad/                    # GridRad NEXRAD .nc 文件
│   ├── nexrad_3d_v4_2_20230102T000000Z.nc
│   ├── nexrad_3d_v4_2_20230102T000500Z.nc
```

### 2.3 NC 文件必需变量

NEXRAD GridRad .nc 文件必须包含以下变量（测试样例已验证）：
- `Longitude` — 经度 (480,)
- `Latitude` — 纬度 (480,)
- `Altitude` — 海拔 (29,)
- `index` — 稀疏索引
- `Reflectivity` — 反射率
- `SpectrumWidth` — 谱宽
- `AzShear` — 方位切变
- `Divergence` — 散度

## 3. 预处理

### 3.1 TUI 方式

```
主菜单 → Ctrl+T（开始训练）→ 进入数据集选择页面

1. 选择 NEXRAD .nc 目录 → Browse → Enter 确认
2. 选择 风暴轨迹 .csv 目录 → Browse → Enter 确认
3. 设置预处理输出目录（默认 ./processed）
4. Ctrl+V → 验证数据格式（通过后显示绿色提示）
5. Ctrl+P → 开始预处理（显示进度条）
6. Ctrl+S → 跳过预处理，进入训练配置
```

### 3.2 CLI 方式（服务器批量预处理）

```bash
python -m trainer_app --mode preprocess \
  --nexrad-dir /data/nexrad/ \
  --tracks-dir /data/tracks/ \
  --hrrr-dir /data/hrrr/          # 可选，HRRR 环境场
  --output-dir ./processed/ \
  --history-steps 12 \
  --future-steps 36 \
  --grid-size 128 \
  --spatial-degree 2.56 \
  --num-workers 4                 # 多线程加速
```

### 3.3 预处理输出

```
processed/
├── storm_1_20230102.npz      # 单窗口样本
├── storm_1_20230102_w0.npz   # 多窗口样本（滑动窗口）
├── storm_1_20230102_w1.npz
├── ...
└── preprocess_report.json    # 预处理报告
```

每个 `.npz` 文件包含：

| 键 | 形状 | 说明 |
|----|------|------|
| `input` | (12, 128, 128, 29, 4) | 历史雷达输入（归一化后） |
| `target` | (36, 128, 128, 29) | 未来涡度标签（归一化后） |
| `target_raw` | (36, 128, 128, 29) | 未来涡度标签（原始值） |
| `storm_uv` | (48, 2) | 风暴运动矢量 |
| `storm_lon_lat` | (48, 2) | 风暴中心坐标 |
| `ef_label` | int | EF 等级 |
| `norm_params` | dict | 归一化参数（含 target_min/target_max） |

### 3.4 预处理参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `history_steps` | 12 | 历史窗口长度（5min×12=1h） |
| `future_steps` | 36 | 预测窗口长度（5min×36=3h） |
| `spatial_degree` | 2.56 | 空间窗口经纬度跨度（约 280km） |
| `grid_size` | 128 | 格点数 H×W |
| `vertical_layers` | null | null=全部29层；数字=只保留前N层 |
| `vertical_stride` | 1 | 垂直降采样步长（1=不降采样） |
| `window_stride` | null | 滑动窗口步长（null=history_steps, 即不重叠） |
| `nan_max_ratio` | 0.3 | 任何层NaN比例>此值则丢弃样本 |
| `gaussian_sigma` | 1.0 | 标签高斯滤波σ值（格点数） |
| `num_workers` | 0 | 多线程数（0=单线程） |

| `Ctrl+P` | 暂停训练 |

| `Ctrl+X` | 终止训练 |
| `Ctrl+B` | 切换到后台运行 |

### 4.2 CLI 方式（服务器批量训练）

```bash

  --processed-dir ./processed/ \
```

### 4.4 训练监控指标说明

训练和验证时，每个 batch 输出以下指标：

| 指标 | 说明 | 期望趋势 |
|------|------|----------|
| **Loss (MAE)** | 涡度预测的 L1 误差 | ↓ 下降 |
| **Total Loss** | 全部 7 项损失的加权和 | ↓ 下降 |
| **Accuracy** | 涡度>0.001 的二分类准确率 | ↑ 上升 |
| **CSI** | Critical Success Index (= IoU) | ↑ 上升 |
| **Dice / F1** | 重叠度 / 调和均值 | ↑ 上升 |
| **Grad Loss** | 涡度梯度幅值误差 | ↓ 下降 |
| **CSI Loss** | 可微 Soft IoU 损失 | ↓ 下降 |
| **FCSI Loss** | 多尺度 FSS 损失 | ↓ 下降 |
| **LPIPS Loss** | 感知相似度（每5 batch） | ↓ 下降 |
| **AUC Loss** | 软 AUC 排序损失 | ↓ 下降 |
| **pred_std / target_std** | 预测/真实涡度标准差 | 比值→1.0 |
| **pos_ratio** | 正类像素比例 | 通常 1-5% |
| **zero_baseline_acc** | 全零预测准确率（模型下限） | 稳定 |

> **CSI 是最关键的指标**：它直接衡量龙卷核心定位能力。IoU/CSI > 0.3 即为良好。

## 5. 推理预测

### 5.1 TUI 方式

```
主菜单 → Ctrl+I → 进入推理页面

1. 选择训练好的 .pt checkpoint 文件
2. 选择预处理后的 .npz 输入文件（或整个目录）
3. 设置预测步数（默认 36 = 3h）
4. Ctrl+R → 开始推理
```

### 5.2 CLI 方式

```bash
# 单文件预测
python -m trainer_app --mode inference \
  --checkpoint ./output/best_model.pt \
  --input ./processed/storm_69.npz \
  --output ./predictions/storm_69_pred.npz \
  --steps 36

# 批量预测（目录下所有 .npz）
python -m trainer_app --mode inference \
  --checkpoint ./output/best_model.pt \
  --input-dir ./processed/ \
  --output-dir ./predictions/

# 带指标输出（需 npz 包含 target）
python -m trainer_app --mode inference \
  --checkpoint ./output/best_model.pt \
  --input ./processed/storm_69.npz \
  --output ./predictions/storm_69_pred.npz \
  --metrics
```

```python
        "mae": 0.0012,
        "csi": 0.35,             # CSI (= IoU)
        "fcsi_mean": 0.52,       # 多尺度 FSS 均值
        "fcsi_1": 0.28,          # FSS at 2.2km
        "fcsi_3": 0.38,          # FSS at 6.6km
        "fcsi_5": 0.48,          # FSS at 11km
        "fcsi_11": 0.61,         # FSS at 24km
        "fcsi_21": 0.85,         # FSS at 46km
        "lpips": 0.32,           # 感知相似度（越小越好）
        "auc": 0.78,             # AUC-ROC
        "f1": 0.41,
        "pos_ratio": 0.03,       # 正类比例
    }
}
```

### 5.4 指标解读指南

| 场景 | CSI | fCSI(3) | fCSI(21) | LPIPS | AUC | 含义 |
|------|-----|---------|----------|-------|-----|------|
| 优秀 | >0.4 | >0.5 | >0.85 | <0.25 | >0.85 | 精确定位+环境识别 |
| 良好 | >0.3 | >0.4 | >0.75 | <0.35 | >0.75 | 可操作预报 |
| 需改进 | <0.2 | <0.3 | <0.6 | >0.5 | <0.65 | 需更多训练 |

- **CSI 低但 fCSI(21) 高** → 模型识别出了风暴区域，但定位有偏差（常见，可接受）
- **fCSI 在所有尺度都低** → 模型基本没学到有效特征

## 6. 输出文件说明

### 6.1 模型架构总结

```
输入 (B,12,128,128,29,4)
  │  雷达数据：Reflectivity, SpectrumWidth, AzShear, Divergence
  ▼
ResUNet 3D 编码器（高分辨率，不下采样）
  │  输出：瓶颈特征 (B,256,128,128,29) + skip connections
  ▼
投影头：ζ_init, velocity(T_out×3), omega(T_out×2)
  │
  ▼
物理模块（平流 + 倾斜拉伸 + 扩散 + 可选斜压）
  │  ∂ζ/∂t = -v·∇ζ + ω·∇w + ν∇²ζ + baroclinic
  ▼
RK2 积分器（36 步推进，每步 300s = 5min）
  │  ζ⁰ → ζ¹ → ... → ζ³⁶
  ▼
ResUNet 3D 解码器（skip 连接 + 上采样）
  │
  ▼
输出 (B,36,128,128,29) — 未来 3 小时涡度预测
```

### 6.2 损失函数

```
L_total = L_mae + 0.1·L_grad + 0.01·L_cont
        + 1.0·L_csi + 0.5·L_fcsi
        + 0.1·L_lpips + 0.05·L_auc
```

### 6.3 关键配置参数

```yaml
# config_default.yaml 核心参数

data:
  history_steps: 12        # 历史窗口（12×5min=1h）
  future_steps: 36         # 预测窗口（36×5min=3h）
  spatial_degree: 2.56     # 空间窗口（2.56°≈280km）
  grid_size: 128           # 格点数
  vertical_layers: null    # null=全部29层
  channel_variables:       # 输入通道
    - Reflectivity
    - SpectrumWidth
    - AzShear
    - Divergence

model:
  base_channels: 64        # 基础通道数
  depth: 2                 # 编码器层数
  encoder_downsample: false # 高分辨率（全128×128）
  dropout: 0.3             # Dropout 正则化
  decoder_chunk_size: 1    # 时间步分块（1=逐步，省显存）
  loss:
    lambda_grad: 0.1       # 梯度损失权重
    lambda_csi: 1.0        # CSI 损失权重
    lambda_fcsi: 0.5       # FSS 损失权重
    lambda_lpips: 0.1      # LPIPS 损失权重
    lambda_auc: 0.05       # AUC 损失权重

training:
  batch_size: 1            # 批次大小
  epochs: 100              # 训练轮数
  learning_rate: 0.001     # 学习率
  use_amp: true            # 混合精度训练
  early_stop_patience: 15  # 早停耐心
```

## 7. 配置参数速查

### 调整指南

| 如果... | 调这个... | 方向 |
|---------|-----------|------|
| GPU 显存不够 (OOM) | `batch_size` ↓ | 2→1 |
| GPU 显存仍不够 | `decoder_chunk_size` ↓ | 4→1 |
| GPU 显存还不够 | `vertical_layers` ↓ | null→10 |
| 模型定位不准 | `lambda_csi` ↑ | 1.0→2.0 |
| 模型结构识别差 | `lambda_fcsi` ↑ | 0.5→1.0 |
| Loss 不收敛 | `learning_rate` ↓ | 0.001→0.0001 |
| 过拟合（train loss 远低于 val） | `dropout` ↑ | 0.3→0.5 |
| 训练太慢 | `num_workers` ↑ | 0→4 |

## 8. 常见问题

### Q: 数据验证失败？

**检查清单**：
1. `.nc` 文件是否包含 `index`、`Reflectivity`、`AzShear` 变量
2. `.csv` 文件是否包含 `Time`、`Longitude`、`Latitude` 列
3. 文件路径不含中文字符（Windows netCDF4 限制，代码已做自动转存处理）
4. 运行 `python -c "import netCDF4; print('OK')"` 确认安装成功

### Q: 训练时 OOM (Out of Memory)？

**依次尝试**：
1. `batch_size: 1`（已默认）
2. `decoder_chunk_size: 1`（逐时间步解码）
3. `vertical_layers: 10`（只用前10层）
4. `grid_size: 64`（降低空间分辨率，同时改 `spatial_degree`）

### Q: 如何从断点继续训练？

```bash
python -m trainer_app --mode train \
  --pretrained ./output/<任务名>/checkpoints/best_model.pt \
  --load-optimizer true
```

或在 TUI 训练配置页面的 "Pretrained Weights" 中选择 checkpoint，勾选 "Load Optimizer State"。

### Q: 训练输出中说 "0 samples" 怎么解决？

这是数据覆盖不足的问题。预处理需要 **同一风暴至少 48 个连续时间步**（2.5 分钟容差内匹配到雷达文件）。确认：

1. NC 文件覆盖了 CSV 中风暴的整个时间段（每 5 分钟一个文件）
2. 风暴轨迹至少持续 4 小时（48×5min）
3. 查看 `preprocess_report.json` 了解确切跳过原因

### Q: 指标中 CSI 一直很低怎么办？

- **正常情况**：龙卷涡度场极其稀疏（正类像素 ~1-5%），CSI > 0.2 已经是有意义的预测
- **检查 zero_baseline_acc**：如果 accuracy ≈ zero_baseline_acc，说明模型没学到任何东西
- **检查 pred_std**：如果 pred_std << target_std，模型预测太保守
- **增加 lambda_focal**：加大 loss 对非零区域的惩罚
