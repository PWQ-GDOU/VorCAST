## 4. 运行预处理

### 4.1 确认配置

打开配置文件确认这几行（一般不需要改，但看一眼）：

```bash
cat ~/QiXiangYuCe/trainer_app/trainer_app/config_default.yaml | grep -A2 "in_channels"
```

**必须是**：`in_channels: 6` 和 `use_storm_motion: true`

### 4.2 启动 TUI 预处理

```bash
cd ~/QiXiangYuCe/trainer_app
source venv/bin/activate
python -m trainer_app.main
```

会看到一个文本界面，操作顺序：

```
1. 主菜单 → 选 「Dataset Setup」
2. 按方向键选择 Nexrad 目录 → 输入 ~/QiXiangYuCe/data/nexrad
3. 按方向键选择 Tracks 目录 → 输入 ~/QiXiangYuCe/data/tracks
4. 按方向键选择 Output 目录 → 输入 ~/QiXiangYuCe/data/processed
5. 返回主菜单 → 选 「Process Events」
6. 等待进度条跑完（可能几十分钟到几小时，取决于数据量）
```

**不想用 TUI？** 一行命令：

```bash

# 复制推理脚本到 workspace
cat > eval_metrics_fss.py << 'EOF'
"""Full metrics evaluation with FSS multi-scale."""
import torch, numpy as np, sys
sys.path.insert(0, 'trainer_app')

from trainer_app.utils.config import load_config
from trainer_app.models.inference import InferenceEngine
from trainer_app.training.monitor import compute_binary_metrics
from trainer_app.models.metrics import soft_fss_loss

config = load_config('trainer_app/config_default.yaml')
engine = InferenceEngine.from_checkpoint(
    'history/run_xxx/checkpoint_best.pt', config  # 替换为实际路径
)

# 加载验证集
import glob
files = sorted(glob.glob('../data/splits/val/*.npz'))
print(f'Valid samples: {len(files)}')

all_metrics = []
for f in files:
    pred, target, _ = engine.predict_from_file(f, compute_metrics=False)
    bm = compute_binary_metrics(pred[0], target[0])
    all_metrics.append(bm)

for k in ['csi', 'dice', 'recall', 'auc', 'f1']:
    vals = [m[k] for m in all_metrics]
    print(f'{k.upper():8s}: {np.mean(vals):.4f} +/- {np.std(vals):.4f}')
EOF

python eval_metrics_fss.py
```

## 8. 配置文件速查

修改 `trainer_app/config_default.yaml` 中的以下项调整训练：

```yaml
data:
  in_channels: 6              # ← 必须是 6（4雷达 + 2风暴运动）
  use_storm_motion: true      # ← 必须是 true
  history_steps: 12           # 输入历史帧数
  future_steps: 36            # 预测未来帧数

model:
  in_channels: 72             # ← 必须是 72（12×6）
  encoder_downsample: false   # 高分辨率模式
  base_channels: 64           # 基础通道数（显存不够可降到 32）
  decoder_chunk_size: 2       # 显存不够可降到 1

training:
  epochs: 150                 # 训练总轮数
  batch_size: 2               # 显存不够可降到 1
  learning_rate: 1.0e-4
  patience: 20                # 早停耐心
```

## 9. 常见报错与解决

| 报错 | 原因 | 解决 |
|------|------|------|
| `CUDA out of memory` | 显存不够 | `batch_size: 1`, `decoder_chunk_size: 1` |
| `no .nc files found` | 数据路径不对 | 检查 `dataset1_path` 指向的目录 |
| `ImportError: No module named 'lpips'` | 依赖没装全 | `pip install lpips` |
| `shape mismatch` | 旧数据用了 4 通道 | 重新跑预处理 |
| `netCDF4: No such file` | 中文路径问题 | 数据必须在纯 ASCII 路径下 |
| `in_channels mismatch` | 配置没更新 | 确认 `data.in_channels=6`, `model.in_channels=72` |
